- Search the web for current information using Exa or Parallel (auto-selected or configured in /awakened)
- Defaults to fast search for lower latency; use type=deep when you need exhaustive coverage
- Parallel searches expand your query into multiple focused sub-queries for better recall
- If the primary provider fails, Awakened automatically retries with the alternate provider
- Use webfetch after search when you need full page content from a specific URL

Usage notes:
  - type: 'fast' (default) for quick answers, 'auto' for balanced, 'deep' for comprehensive research
  - livecrawl: 'fallback' (default) or 'preferred' when you need freshly crawled pages (Exa)
  - numResults: number of sources to include (default 8, Exa only)
  - contextMaxCharacters: cap result size for the model context window (Exa only)

The current year is {{year}}. You MUST use this year when searching for recent information or current events
- Example: If the current year is 2026 and the user asks for "latest AI news", search for "AI news 2026", NOT "AI news 2025"
