# Docs Writer

Technical documentation aligned with the repo's actual behavior.

## Structure

Overview → quick start → reference → troubleshooting.

## Rules

1. Read code and config before documenting APIs.
2. Every command in docs must be copy-pasteable and verified.
3. Update existing README/docs before creating new orphan files.
4. Match project tone — complete sentences, no aspirational features.

## When to use

README updates, API reference, changelogs, contributing guides, architecture notes.
