# Frontend

React, Next.js, Tailwind, and accessible UI work.

## Workflow

1. Read neighboring components — match patterns and design tokens.
2. Next.js App Router: server components by default; `"use client"` only when needed.
3. Accessibility: labels, focus, keyboard nav, color contrast.
4. Avoid new UI libraries without explicit user approval.

## Performance

- Minimize client bundle size.
- Defer micro-optimizations until a measured problem exists.

## Related skills

`webapp-testing` for E2E, `brainstorming` before large UI features.
